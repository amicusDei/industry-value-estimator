Modify `scripts/run_statistical_pipeline.py` to wire all four orphaned function groups. Follow the RESEARCH.md target pipeline structure exactly. The changes are:

**1. New imports (add after existing project imports):**
```python
from src.diagnostics.structural_breaks import run_cusum, run_chow
from src.processing.features import assess_stationarity
from src.models.statistical.regression import fit_top_down_ols_with_upgrade
import statsmodels.api as sm
```

**2. New function `_load_lseg_scalar()` (add after `_load_real_data`):**
- Load `DATA_PROCESSED / "lseg_ai.parquet"`. If file missing, return empty dict (pipeline runs without LSEG).
- Group by `industry_segment`, sum `Revenue`, divide by 1e9, then normalize each segment's value as share of total revenue.
- Return `dict[str, float]` mapping segment name to revenue share in [0, 1].
- Log what was loaded (segment names and scalar values).

**3. Modify `_build_segment_series` signature:**
- Add parameter: `lseg_scalar: dict[str, float] | None = None`
- After `build_pca_composite` call, if `lseg_scalar` is not None and `segment` is in `lseg_scalar`, multiply `scores` by `(1.0 + lseg_scalar[segment])` — a gentle amplification per RESEARCH.md Pattern 1.
- Print the scalar value when applied.
- This is backward-compatible: existing callers that don't pass lseg_scalar get unchanged behavior.

**4. New function `_run_break_detection(combined_series: pd.Series) -> int`:**
- Accept a pd.Series indexed by integer year.
- Run `run_cusum(combined_series)`.
- Calculate `break_idx` for year 2022 in the series index. Guard: if break_idx < 3 or break_idx > len(series)-3, skip Chow and return 2022 (Pitfall 3 from RESEARCH.md).
- Run `run_chow(combined_series, break_idx=break_idx)`.
- Print CUSUM p-value and Chow F-stat/p-value.
- If Chow p < 0.05, return `int(chow["break_year"])`. Elif CUSUM p < 0.05, return 2022. Else return 2022 (default fallback).
- Print the detected break year.

**5. Modify `run_pipeline()` function — real data path (inside `if use_real_data:` block):**

After `_load_real_data()` and before the segment loop:
- Call `lseg_scalar = _load_lseg_scalar()` and print the result.
- Build an aggregate series for break detection: average PCA composite across all segments (or use the first segment's series). Simplest: build the ai_software segment series first (it has the most data including LSEG), then pass it to `_run_break_detection`. Store result as `break_year`.
- Thread `lseg_scalar` into `_build_segment_series(combined, seg, lseg_scalar=lseg_scalar)` calls.

Inside the per-segment loop (for BOTH real and synthetic paths), BEFORE `select_arima_order()`:
- Call `assess_stationarity(series.values)` and print the result (ADF p-value, KPSS p-value, recommended d). Add a note if N < 20 per Pitfall 4.

Inside the per-segment loop, after winner selection and residual extraction (BEFORE appending to summary_rows):
- Call `fit_top_down_ols_with_upgrade(series.values, sm.add_constant(np.arange(len(series))))` — use trend index as X since GDP-share requires GDP series which is only available in real-data mode. For synthetic mode, use time trend as X.
- For real-data mode: if `combined` has `gdp_real_2020_usd`, align it with the segment series index and use GDP values as X.
- Wrap in try/except (OLS may fail on very short series) — on failure, print warning and continue.
- Print model_type and R-squared from the diagnostics dict.

Thread `break_year` (default 2022 for synthetic mode) into `run_prophet_cv(df, seg, n_splits=n_splits, changepoint_year=break_year)` and `fit_prophet_segment(df, seg, changepoint_year=break_year)`.

**6. Update module docstring** to reflect the four new integrations.

IMPORTANT: Do NOT change the residuals output schema — OLS diagnostics are logged only, not written to residuals_statistical.parquet (per RESEARCH.md Open Question 3).

IMPORTANT: The synthetic data path (`use_real_data=False`) should also call `assess_stationarity` and `fit_top_down_ols_with_upgrade` per-segment (so tests work). For break detection and LSEG scalar, these are only relevant for real data — set `break_year = 2022` and `lseg_scalar = None` in synthetic mode.
