**Step 1: Create data/raw/market_anchors/ directory and ai_analyst_registry.yaml**

Create the directory `data/raw/market_anchors/` and the YAML registry file. The registry follows this schema per entry:

```yaml
entries:
  - source_firm: "IDC"
    report_name: "Worldwide AI Spending Guide 2024"
    publication_year: 2024
    estimate_year: 2023
    segment: "total"
    as_published_usd_billions: 207.0
    currency: "nominal_usd"
    scope_includes: "enterprise AI software, IT infrastructure, and services"
    scope_excludes: "consumer AI applications, general IT not AI-specific"
    scope_segment_mapping:
      ai_hardware: 0.0
      ai_infrastructure: 0.40
      ai_software: 0.35
      ai_adoption: 0.25
    methodology_notes: "Vendor survey + secondary research. Narrow enterprise definition."
    source_url: "https://www.idc.com/..."
    confidence: "high"
```

Populate with real published estimates. Target: 40-60 entries total covering:
- 6-8 analyst firms (IDC, Gartner, Grand View Research, Statista, Goldman Sachs, Bloomberg Intelligence, McKinsey, CB Insights)
- Years: 2017-2025 (not every firm covers every year)
- Segments: primarily "total" entries, with per-segment breakdown entries where published
- Vintage series: multiple publication_years for the same estimate_year where available (e.g., IDC's 2023 estimate as published in 2023 vs 2024)

Use real published figures where known from research context. For entries where exact figures are not available in the research materials, use reasonable estimates based on known market size range ($150-250B for narrow scope 2023, $500B-1.5T for broad scope 2025) and mark confidence as "medium" or "low". Every entry MUST have all required fields -- no optional fields, no nulls.

    Full compilation pipeline:

    6. Return compiled DataFrame (do NOT write Parquet yet -- Plan 08-04 handles that)
    """
