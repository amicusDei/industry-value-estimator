# Phase 4 — UI Design Contract

> Visual and interaction contract for the Plotly Dash interactive dashboard.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Plotly Dash 4.0 — Python component tree, not React design system) |
| Preset | not applicable |
| Component library | Plotly graph objects (go.*) + Dash HTML/core components; dash-bootstrap-components 2.0.4 for layout grid |
| Icon library | none (text labels only; no icon library needed) |

**shadcn gate result:** Not applicable. Project is Python/Dash — no React, no Tailwind, no components.json. CSS is authored inline via Dash `style={}` dicts and the `assets/` directory.

## Spacing Scale

All spacing values expressed as px integers for use in Dash `style={}` dicts.
Must be multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, attribution footnote top margin, compact badge padding |
| sm | 8px | Between chart and attribution footnote, between label and control |
| md | 16px | Default card internal padding, column gap between chart panels |
| lg | 24px | Section padding within tabs, space between headline stat and fan chart |
| xl | 32px | Gap between major layout regions (header → tab content) |
| 2xl | 48px | Vertical breathing room at top/bottom of tab content area |
| 3xl | 64px | Not used in this phase — page is scrollable dashboard, not landing page |

Exceptions:
- Attribution `html.P()` top margin: 4px (`xs`) — intentionally tight to its chart
- Global controls header bottom margin: 8px (`sm`) — compact, controls are utility
- Headline number bottom margin: 24px (`lg`) before the fan chart

## Typography

All sizes expressed as px integers for use in Dash inline `style={"fontSize": "Npx"}`.
Two weights only: regular (400) and semibold (600).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 36px | 600 | 1.2 | Headline stat ("AI Industry: $X.X Trillion by 2030") on Overview tab |
| Heading | 20px | 600 | 1.2 | Tab-section headings (e.g. "Forecast Fan Chart", "Segment Breakdown") |
| Body | 16px | 400 | 1.5 | General text, diagnostics table cell values, tooltip copy |
| Label | 12px | 400 | 1.4 | Attribution footnotes, axis tick labels override, scorecard metric row labels |

Attribution footnote is 11px (sub-label) — sole exception below the label tier, reserved exclusively for DATA-07 source text below charts. Source: RESEARCH.md Pattern 4.

## Component Inventory

This phase uses Dash primitives — no shadcn, no third-party component registry. All interactive elements are standard Dash core components.

| Component | Dash Element | Location |
|-----------|-------------|----------|
| Global segment filter | `dcc.Dropdown` | Header bar, persists across all tabs |
| USD toggle | `dcc.RadioItems` (inline) | Header bar, right of segment dropdown |
| Tab navigation | `dcc.Tabs` + `dcc.Tab` | Below header bar |
| Fan charts | `go.Figure` via `dcc.Graph` | Overview, Segments tabs |
| Segment breakdown bar chart | `go.Figure` via `dcc.Graph` | Overview tab |
| SHAP attribution | `html.Img(src="/assets/shap_summary.png")` | Drivers tab — static PNG, no re-render |
| Diagnostics scorecard | `html.Table` | Diagnostics tab — 4-segment comparison |
| Backtest chart | `go.Figure` via `dcc.Graph` | Diagnostics tab |
| Attribution footnotes | `html.P()` | Below every `dcc.Graph` |
| Layout grid | `dbc.Row` / `dbc.Col` (dash-bootstrap-components 2.0.4) | Tab content areas |
| Loading indicator | `dcc.Loading` wrapping each `dcc.Graph` | Wrap all charts; type="circle" |

Decision to use dash-bootstrap-components: The 4-tab layout with side-by-side chart panels warrants a proper grid system. Plain `html.Div` + flexbox inline styles produce the same result but increase maintenance burden. DBC 2.0.4 is already listed as a discretionary dependency in RESEARCH.md and is confirmed available on PyPI.

## Interaction Contract

### Global Controls (header, visible on all tabs)

| Control | Type | Default | Behavior |
|---------|------|---------|----------|
| Segment dropdown | `dcc.Dropdown` | `"all"` = "All Segments" | Filters all charts on the current active tab simultaneously. Never resets on tab switch — placed outside tab content container. |
| USD toggle | `dcc.RadioItems` | `"point_estimate_real_2020"` | Switches y-axis column used in all fan charts and headline stat. All charts update simultaneously. |

### Callback Architecture

Single tab-level callback: `(main-tabs.value, segment-dropdown.value, usd-toggle.value) → tab-content.children`. Re-renders entire active tab layout on any input change. Per-graph callbacks introduced only if visual flicker is unacceptable during testing (RESEARCH.md open question 3).

### Chart Interactions (Plotly default behaviors — do not suppress)

| Behavior | Keep? | Note |
|----------|-------|------|
| Hover tooltips | Yes | Compact template: `"<b>%{x}</b><br>%{y:.2f}T USD<extra></extra>"` |
| Zoom / pan | Yes | Standard Plotly zoom — no suppression |
| Download PNG button | Yes | Default Plotly modebar — keep; useful for portfolio sharing |
| Legend click to hide trace | Yes | Allows user to isolate individual CI bands or segments |
| Double-click to reset zoom | Yes | Standard Plotly behavior |

### Fan Chart Visual Contract

Trace rendering order (bottom to top — first added renders lowest):
1. 95% CI band (`fill="toself"`, `rgba(30,90,200,0.10)`) — widest, most transparent
2. 80% CI band (`fill="toself"`, `rgba(30,90,200,0.20)`) — narrower, slightly more opaque
3. Historical line (solid, `#1E5AC8`, width=2)
4. Forecast line (dashed, `#1E5AC8`, width=2) — bridged from last historical point
5. Vertical dashed boundary line via `fig.add_vline(x=2024, ...)`

Historical region: white background (`plot_bgcolor="#FFFFFF"`)
Forecast region: light tinted background — `fig.add_vrect(x0=2024, x1=2030, fillcolor="rgba(244,246,250,0.6)", layer="below", line_width=0)` — belt-and-suspenders visual clarity per CONTEXT.md decision.

### Tab Content Overview

| Tab | Key Components | Empty State |
|-----|---------------|-------------|
| Overview | Headline stat + fan chart (all segments aggregate) + segment bar chart | "No forecast data loaded. Check that forecasts_ensemble.parquet exists at data/processed/." |
| Segments | Grid of per-segment fan charts (2×2 layout via dbc.Row/Col) | Same as Overview |
| Drivers | SHAP PNG + explanatory text + attribution footnote | "SHAP summary plot not found. Verify assets/shap_summary.png exists." |
| Diagnostics | Metrics scorecard table + per-segment backtest chart | "Diagnostics unavailable. Check that residuals_statistical.parquet exists at data/processed/." |

### Loading State

Wrap every `dcc.Graph` with `dcc.Loading(type="circle", color="#1E5AC8")`. Loading spinner uses accent primary color. No skeleton displays — data loads at module level (near-instant after startup). The loading state covers the brief re-render during callback execution when the user switches segments or toggles USD.
